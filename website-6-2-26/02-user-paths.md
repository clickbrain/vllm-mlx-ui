# Kilroy User Path Document (Interest → Action)

## 1) End User Journey

### Entry Triggers
- Looking for simpler AI automation
- Wants private, personal control over apps and feeds
- Frustrated by fragmented tools

### Path Flow
1. **Home hero** communicates no-code, desktop-first control
2. **End User card click** routes to practical benefits page
3. **Week-one outcomes** reduce complexity anxiety
4. **Simple onboarding sequence** makes first success obvious
5. **Action CTA** pushes to download/quick-start/contact

### Trust Builders
- Plain language
- Concrete day-1 actions
- Privacy and local-control messaging

### Primary Conversion Action
- Start with quick start / download interest

---

## 2) Enterprise / Business Journey

### Entry Triggers
- Team productivity bottlenecks
- Need for automation with data control
- Interest in private collaboration and controlled deployment

### Path Flow
1. **Home card** routes into enterprise outcome framing
2. **Outcome section** ties Kilroy to speed, ownership, integration
3. **Governance/deployment section** answers risk and control concerns
4. **Pilot blueprint** gives low-friction adoption model
5. **Action CTA** leads to discovery call request

### Trust Builders
- Architecture language over hype
- Pilot-first structure
- Clear mention of deployment options

### Primary Conversion Action
- Book enterprise discovery / request architecture brief

---

## 3) Developer Journey

### Entry Triggers
- Wants to build custom AI-enabled workflows
- Wants to combine visual building with code-level control
- Needs packaging/distribution model for apps

### Path Flow
1. **Home card** routes to developer page
2. **Build value section** presents extensibility and architecture
3. **Build journey** outlines practical path from app creation to integration
4. **Workflow/webhook references** reinforce technical credibility
5. **Action CTA** routes to docs and first-build action

### Trust Builders
- Real concepts from docs (manifest, workflows, wf_agents)
- Sequence-oriented builder flow
- Fast path to implementation docs

### Primary Conversion Action
- Start first app / visit docs

---

## 4) Investor Journey

### Entry Triggers
- Seeking platforms at intersection of AI, automation, and decentralization
- Looking for product + ecosystem leverage
- Wants evidence of multi-segment monetization potential

### Path Flow
1. **Home card** routes to investor narrative page
2. **Market thesis** defines urgency and opportunity
3. **Leverage points** show product, developer, and ecosystem flywheel
4. **Milestone narrative** translates technical traction to business narrative
5. **Action CTA** requests deck/founder meeting

### Trust Builders
- Clear market framing
- Segment-based growth logic
- Balanced, non-overclaiming language

### Primary Conversion Action
- Request investor deck / schedule discussion

---

## Subtle Cross-Path Transitions (Natural Routing)

To avoid hard segmentation and increase conversion quality:
- End users are shown a "building custom workflows?" handoff to developer page
- Enterprise page includes "need technical implementation details?" handoff to developer page
- Developer page includes "bringing this to a team?" handoff to enterprise page
- Investor page includes links to platform and developer pages for diligence

This keeps each audience in its main lane while creating organic movement as intent matures.
