# Kilroy Website Structural Outline (Pages + Core Content)

## 1) Home (`index.html`)
**Goal:** Explain Kilroy in plain language, then route visitors naturally by intent.

**Sections:**
- Hero: "Build your personal internet. No code required."
- Core value bar: no-code creation, private swarms, app ecosystem, desktop-first control
- "Choose your path" cards (subtle audience routing):
  - End Users
  - Enterprise / Business
  - Developers
  - Investors
- "How Kilroy works" (Apps + Pipelines + Swarms)
- Social proof style product capability list (from docs)
- Primary CTA row: Start using, Explore platform, Contact team

## 2) Platform Overview (`platform.html`)
**Goal:** Give one shared product understanding before deeper segment pages.

**Sections:**
- What Kilroy is (desktop application + decentralized runtime)
- Key concepts:
  - Apps (signed self-contained packages)
  - Pipelines (visual agent composition)
  - Swarms (peer-to-peer encrypted communication)
- Benefits by outcome:
  - Faster creation
  - Private collaboration
  - Portable architecture (desktop and Docker-request deployments)
- CTA row: Try quick start, Build an app, Discuss enterprise needs

## 3) End Users Path (`end-users.html`)
**Goal:** Convert curious users into active users.

**Sections:**
- "What you can do in week one"
- Benefits:
  - Centralized personal dashboard
  - AI-assisted workflows without code
  - Private peer-to-peer sharing
- Onboarding path:
  - Download
  - Install
  - Add first app
  - Run first pipeline
- CTAs: Download interest form, See quick start, Join community/contact

## 4) Enterprise / Business Path (`enterprise.html`)
**Goal:** Convert operations/innovation leaders into pilot conversations.

**Sections:**
- Business outcomes:
  - Internal automation acceleration
  - Data/control ownership
  - Multi-system integration through app/pipeline model
- Governance and deployment positioning:
  - Local-first architecture
  - Team-controlled sharing via private swarms
  - Headless/Docker option by request
- Pilot blueprint:
  - Discovery
  - Pilot workflow
  - Scale plan
- CTAs: Book discovery call, Request architecture brief

## 5) Developer Path (`developers.html`)
**Goal:** Convert builders into app/pipeline contributors.

**Sections:**
- Why build on Kilroy:
  - Visual + JavaScript extensibility
  - App packaging model
  - Workflow/webhook patterns
- Build journey:
  - Create app scaffold
  - Add process diagrams/workflows
  - Integrate wf_agents and webhooks
  - Package and distribute
- CTA stack: Read docs, Build first app, Contact for advanced integration

## 6) Investor Path (`investors.html`)
**Goal:** Convert investor curiosity into qualified conversations.

**Sections:**
- Market thesis framing:
  - AI + automation usability gap
  - Privacy/decentralization demand
  - Emerging app economy for composable agents
- Product leverage points:
  - No-code adoption layer
  - Developer extension layer
  - Distribution/app marketplace potential
- Milestone narrative:
  - Product readiness indicators
  - Ecosystem growth vectors
- CTAs: Request investor deck, Schedule founder conversation
